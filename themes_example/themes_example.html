Skeleton Themes Example

This example goes through the differing things with the Skeleton Themes implementation. Everything else is the same, and is further described at getskeleton.com.

You can select one of the template themes using the dropdown menu to the right.

Theme Elements

A Skeleton theme is defined by a set of variables that can be exchanged in order to change theme. These variables are defined in their own themes class which can be applied to a parent element which contains everything you want the theme to affect. This site does this by setting the class of the body element to one of these themes, and exchanging that class when you change theme.

A theme is composed of different sets of variables that applies to different things. For a start we can look at the definition of the Skeleton Blue theme, which is the original look. 

                
/* Classic bone chilling blue */

.skeleton-blue {
    /* Colors used for primary elements such as primary buttons and links */
    --primary: #1EAEDB;
    --primary-focus: #0FA0CE;
    --primary-background: #33C3F0;
    --primary-text: #FFF;

    /* The color of non-primary elements such as standard buttons */
    --muted: #555;
    --muted-border: #bbb;
    --muted-focus: #333;
    --muted-border-focused: #888;

    /* Text color on page for all elements */
    --text: #222;

    /* Page background color and highlights */
    --background: #FFF;
    --background-highlight: #E1E1E1;

    /* Code blocks */
    --code: #222;
    --code-background: #F1F1F1;
    --code-border: #E1E1E1;

    /* Forms */
    --form-background: #FFF;
    --form-border: #D1D1D1;
}

We'll go through the groups one by one.

The --primary group defines the primary color of accents on the site. For the Skeleton Blue theme this is the light blue color. These colors applies to primary buttons and links. Note that there also is a definition for the text color of primary elements, such as the text color of primary buttons.

The --muted group defines how normal elements should look, such as normal buttons. This also includes how the elements should look when they are hovered over.

The --text definition is for how overall text should look in the theme. This is usually black or white, but can be changed as you please.

The --background group defines what color the background should be, and some accents such as horizontal lines.

We'll carry on with these explanations in their corresponding section.

Buttons

Buttons can take the form of a primary or normal button. The primary buttons follow the themes colorscheme for primary elements. The normal buttons follows the themes colorscheme for muted elements.

Links

Links also follows the --primary color definitions.

Skeleton repository on Github

Forms

Forms work the same as they always have, but now things such as borders and buttons follows the color scheme. The background and border colors of forms are defined in the --form group.

Code

Code styling is the same as before; just wrap inline code in a <code> element, and wrap that in a <pre> element to get a code block.

Code can be stylized by changing the color of the text, the background color and the border around it. These colors are defined in the --code group.
